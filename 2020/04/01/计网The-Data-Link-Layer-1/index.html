计网The Data Link Layer(1)

发表于 2020-04-01 | 更新于 2020-04-02 | 分类于 计算机网络

Why do we need Data Link Control?

  • Probable transmission bit errors in physical layer：error control
  • Receiver may need to prevent too much data coming in: flow control
  • Broadcast channel: addressing, media access control

Position and Functions

  • Data link layer has responsibility of transferring data (frame) from one node to adjacent node in a reliable and efficient way over a link

1.Error control:差错检验

2.Flow control:流量控制

3.Addressing:寻址

4.Packetizing:封装成帧

网络层来的数据需要封装成帧，加头加尾，用以实现以上各种功能。

Data Link Layer Design Issues

  • Goals
    • Study the algorithms for achieving reliable, efficient communication between two adjacent machines
  • Data Link Layer Design Issues
    • Services Provided to the Network Layer
    • Framing
    • Error Control: Dealing with transmission errors
    • Flow Control: Slow receivers not swamped by fast senders

Functions of Data Link Layer: Illustration

Packets vs. Frames

Data Link Layer: hop-to-hop communication(逐跳通信)

Data Link Layer: Adapters Communicating

  • Sending side:
    • Encapsulates network packet in a frame
    • Adds error checking bits, reliable transfer, flow control, etc.
  • Receiving side
    • Looks for errors, reliable transfer, flow control, etc
    • Extracts packet, passes to receiving node
  • Link layer implemented in “adapter” (NIC)
    • Ethernet card, PCMCIA card, 802.11 card

Services Provided to Network Layer（1）

链路层是虚拟通信，数据带有协议，不是普通的物理层

Services Provided to Network Layer(2)

  • Connectionless services
    • Unacknowledged connectionless service
      • No acknowledgement, no logical connection
      • Most LANs use such service
    • Acknowledged connectionless service
      • Each frame acknowledged, used in unreliable channel, such as wireless systems
  • Acknowledged connection-oriented service
    • A connection established, frames are numbered, reliable transmission guaranteed, eg. ATM

提供的三种服务，从可靠度来讲，从上到下，越来越可靠；从复杂度来讲，从上到下，越来越复杂。

1.第一种服务有两个关键词，‘无确认’，‘无连接’，就一个数据传输的阶段，快速，简洁，没有确认

无连接服务一般是一个地址的全地址。

适用场合：

1.信道非常可靠，传输不可能出错，比如光纤，几乎零误码

2.近距离通信，LAN，由于距离很近，所以误码率极低

其实总结来说，就是误码率很低的场合。

另外还有一种场合比较适用，对实时性要求很高，语音电话，视频直播，即：对实时性的追求高于误码率，对时延和带宽非常敏感，要求很高，但是对误码率要求不高，在这种情况下可以通过牺牲一定的可靠性来换取高实时性。

2.有确认的无连接服务。信道质量不足够好，eg:wifi，每发一个包一定要确认对方一个ACK，如果没有收到ACK，则会再次重发。这种服务能保证数据能够到达对方，但是不保证数据有序，不重复

3.可靠度最高的服务。通信分为三个阶段。所谓的可靠性是指：不丢失，不重复，无差错，按序。eg:因特网的骨干网，长途电话网，ATM网，低轨卫星

只要是有确认的服务，一般都要求是双向信道，双向通信
