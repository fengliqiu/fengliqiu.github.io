Time Division Multiplexing

  • Codec：digitizing voice signals

    • 8000/s, T=1/8000=125μsec, C=64kbps

    采样频率为8000次每秒，周期为125μsec，如果是8bit的编码，那么一路的PCM信号经过脉冲编码调制之后，4kHz的语音信号如果不经过压缩，那么就会变为64kbps.

  • Multiplexing

    • T1 —-1.544 Mbps 24路PCM信号,8kbps for signaling
    • E1 —-2.048Mbps 30/32路PCM信号

    我国使用的是E1标准，欧美一般使用T1标准

Multiplexing T1 streams into higher carriers.

*Circuit Switching vs. Packet Switching（重点内容）

电路交换和分组交换

  • 电路交换主要适用于PSTN网络，现在用的很少，适用于电话之间的通信。
  • 分组交换适用于计算机之间的通信，用的比较广泛。

Circuit switching（电路交换）

  • Example: Telephone network
  • Dedicated communication path between 2 stations
  • Resources allocated in advance
  • 3 phases: circuit establishment, data transfer and circuit termination
  • 一旦打通，属于专用线路，路径，资源独享
  • 资源需要预先分配，连接建立时分配的
  • 电路交换的三个步骤：电路建立（拨号），数据交换，电路的释放和终止

核心的两点：

1.资源预先分配，是面向连接的，如果没有连接建立是无法预先分配资源的

2.通信路径是专享的，独占的

Demo of Circuit Switching

Message Switching（报文交换）

  • Example：Post office
  • Message transmission and switching
  • At each node the entire message is received, stored, and then forwarded to the next node (Store-and-Forward，存储-转发)

缺点：报文忽大忽小，而且需要占缓冲区的资源，造成交换节点过大，不好调度资源，排队时延是一个重大问题

Packet Switching（分组交换）

 Data sent as formatted bit-sequences (Packets)

 Each packet is independently routed

 At each node the entire packet is received, stored briefly, and then forwarded to the next node (Storeand-Forward，存储-转发)

 Dynamic resource allocation  Each packet is independently routed

Basic Operation of Packet Switching

  • Data transmitted in small packets
    • Longer messages split into series of packets(reduce delay)
    • Each packet contains a portion of user data plus
  • Control information
    • Routing (addressing) information
  • Packets are received, stored briefly (buffered) and past on to the next node
    • Store and forward

分组交换把大报文转换成小组，分组打得细会提高信道的利用率，但是每个组都要写地址，比较麻烦，另一方面，分组之后还要标记哪个小组是第一个，哪个小组是第二个，因为接收方会重组。

因此，分组交换换取了信道的利用率，性能，降低了等待时延，但是增加了复杂度，接收方和发送方都增加了复杂度，相当于用处理的复杂度来换取性能。

Packet Switching: Good & Bad

  • Strength
    • Line efficiency
      • Single node to node link can be shared by many packets over time
      • Packets queued and transmitted as fast as possible
    • Data rate conversion(速率适配)
      • Each station connects to the local node at its own speed
      • Nodes buffer data if required to equalize rates
    • Packets are accepted even when network is busy(排队制)
      • Delivery may slow down
      • Priorities can be used
  • Weakness
    • Complex network
    • Larger delay
    • No QoS guarantee

1.线路利用率高，共享信道，共享链路资源

2.只要是存储转发的交换都可以进行速率适配，设置缓冲区

3.优先队列，视频通信

1.软件复杂，处理复杂

2.延迟增大（相对于电路交换而言），电路交换只有传播时延和发送时延，但是分组交换和报文交换还有排队的时延。。。报文交换的时延是最长的。。。

3.没有QoS的保障，大家共享带宽，带宽是动态分配的。

Circuit switched and Packet-switched: Comparison

电路交换需要呼叫建立，分组交换的连接建立的不是物理连接，计算时延的时候可以忽略不算。

电路交换的带宽是固定的，64kbps，而分组交换的带宽是不确定的，动态分配的。

最根本的原因是电路交换不是存储转发的交换，在交换节点不处理数据，直接转走，就相当于是一个物理接口。而分组交换在交换节点需要进行处理，在每个交换节点内部可以对数据进行诸如：排队，分优先级，匹配速率的处理，所以分组交换是存储转发的。

Physical Interface Features

  • Mechanical features(机械特性):指明接口所用接线器的 形状和尺寸、引线数目和排列、固定和锁定装置等
  • Electrical features(电气特性):指明在接口电缆的各条线 上出现的电压的范围
    • Encoding, Data rate, cable length
  • Functional features(功能特性):指明某条线上出现的某一 电平的电压表示何种意义
  • Procedure features(过程特性)：指明对于不同功能的各 种可能事件的出现顺序
